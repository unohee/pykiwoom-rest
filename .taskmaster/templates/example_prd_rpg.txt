# 저장소 계획 그래프(RPG) 방식 PRD 템플릿

이 템플릿은 기능 의미와 코드 구조를 분리해 생각한 뒤 명시적 의존성으로 연결합니다.

## 핵심 원칙

1. 기능과 구조를 따로 분석한 뒤 대응시킵니다.
2. 모든 의존성을 명시하고 순환 의존성을 금지합니다.
3. 기반 계층부터 위상 순서대로 구현합니다.
4. 넓은 기능에서 검증 가능한 원자 작업으로 점진적으로 구체화합니다.

## 사용 방법

- 각 `<instruction>` 블록의 질문에 답합니다.
- `<example>`은 좋은 경계와 나쁜 경계를 구분하는 참고입니다.
- 코드 식별자와 경로는 실제 저장소를 조사한 뒤 적습니다.
- 완성한 PRD는 `task-master parse-prd`로 의존성 있는 태스크 그래프로 변환합니다.
</rpg-method>

│   │   └── __init__.py      # 공개 인터페이스
│   └── [module-name]/
├── tests/
└── docs/

<task-master-integration>
# Task Master 변환 규칙

`task-master parse-prd <file>.txt`를 실행하면 다음과 같이 해석합니다.

1. `### 기능 영역`은 상위 태스크가 됩니다.
2. `#### 기능`은 해당 영역의 하위 태스크가 됩니다.
3. 의존성 목록은 태스크 의존성으로 변환됩니다.
4. 단계 번호는 우선순위와 실행 순서에 반영됩니다.
5. 테스트 시나리오는 구현과 검증 맥락으로 사용됩니다.

## 좋은 결과를 위한 점검

1. 의존성 그래프에 충분한 시간을 씁니다.
2. 각 기능을 독립적으로 테스트 가능한 크기로 만듭니다.
3. 넓은 기능은 `task-master expand`로 점진적으로 분해합니다.
4. 조사 모델을 사용할 때도 현재 코드와 공식 문서로 결과를 검증합니다.
</task-master-integration>
