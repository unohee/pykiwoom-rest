# 리팩터링 보고서 기록

## 문서 성격

이 파일은 2026-01-04에 수행한 과거 정적 분석의 기록입니다. 당시 존재했던 파일과 수치는 이후
리팩터링으로 달라졌으므로 현재 코드 상태의 판정 근거로 사용하지 않습니다. 현재 상태는
`cxt audit`, `ruff`, 테스트, 최신 OpenSwarm 리뷰 결과로 다시 측정해야 합니다.

## 당시 주요 관찰

- 사용되지 않는 레거시 구현과 중복 파사드 후보가 있었습니다.
- 차트 날짜 범위 처리와 페이지네이션에 중복이 있었습니다.
- `get_minute_chart_with_date`, `make_tr_request` 등 일부 메서드의 순환 복잡도가 높았습니다.
- 요청 속도 제한 책임이 여러 모듈에 분산되어 있었습니다.

## 현재 재검증 명령

```bash
source ~/dev/mlx_env/bin/activate
cxt audit
cxt check --high-risk
ruff check src tests
pytest -q
```

## 리팩터링 원칙

1. 삭제 전에 현재 호출자와 공개 import를 확인합니다.
2. 날짜 범위·페이지네이션 같은 반복 로직은 의미가 같은 경우에만 공통화합니다.
3. 공개 메서드 시그니처를 바꾸면 CLI, MCP, 테스트, 문서를 함께 갱신합니다.
4. 복잡도 감소 자체보다 API 계약과 오류 의미 보존을 우선합니다.
5. 과거 보고서의 줄 수와 점수를 현재 수치로 재사용하지 않습니다.

## 완료 판정

리팩터링은 파일이 짧아졌다는 이유만으로 완료되지 않습니다. 전체 테스트, 관련 통합 경계,
정적 검사, 리뷰에서 회귀가 없음을 확인해야 합니다.
